# Copilot CLI

This page documents the GitHub Copilot CLI: quick usage notes, common commands and options, and where to find the full documentation on GitHub.

> Note: The Copilot CLI (copilot-cli) evolves quickly. The commands and flags shown here are a concise quick-start — always check the official project README/releases for the latest installation instructions and flags: https://github.com/github/copilot-cli

## What is the Copilot CLI?

The Copilot CLI is a command-line interface that brings GitHub Copilot functionality to your terminal. It provides an interactive chat, code generation and explanation features, and other productivity helpers right from the command line.

This page helps you get started quickly.

## Prerequisites

- A GitHub account with access to GitHub Copilot (Copilot may require a subscription).
- A terminal (macOS, Linux, or Windows with WSL).
- (Optional but recommended) the `gh` GitHub CLI installed and authenticated — some flows may integrate with `gh` for authentication.

## Install

There are multiple installation options. Always prefer the official release for your platform from the project's Releases page on GitHub.

1. Download a released binary from the GitHub Releases page
   - Visit: https://github.com/github/copilot-cli/releases
   - Download the binary for your OS and place it on your PATH (for macOS/Linux you may put it in `/usr/local/bin` or `~/bin`).

2. Homebrew (macOS/Linux) — if a formula is available

   Example (if available):

   ```bash
   brew install copilot-cli
   # or (if provided by the project):
   brew install github/copilot-cli/copilot-cli
   ```

3. Build from source

   Follow the instructions in the project repository to build locally (requires Go/Node toolchain depending on the implementation at the time).

After installation, verify the CLI is available:

```bash
copilot --help
# or
copilot version
```

## Authenticate

The Copilot CLI requires you to authenticate against GitHub to use Copilot features. Typical options:

- Use the built-in login flow the CLI provides (e.g. `copilot login`), which opens a browser and prompts you to authorize.
- If the CLI integrates with `gh`, authenticate with `gh auth login` first and the Copilot CLI will reuse that token.

If you run a command and are prompted to authenticate, follow the instructions shown in the terminal.

## Quick commands (common flows)

Note: Command names and flags may change over time. The examples below show common patterns and are intentionally simple.

- Show help and available commands:

```bash
copilot --help
copilot help <command>
```

- Start an interactive Copilot chat session:

```bash
copilot chat
```

Inside the chat you can ask natural language questions about your codebase, request snippets, or ask for explanations.

- Ask a one-off question (non-interactive):

```bash
copilot chat "How can I improve this Go HTTP handler to handle timeouts?"
```

- Generate code from a prompt:

```bash
copilot code "Create a simple Go HTTP server with a /ping endpoint that returns JSON {\"pong\": true}\nInclude graceful shutdown."
```

- Explain a file or function (example command names may vary):

```bash
copilot explain ./internal/handlers/ping.go
# or
copilot explain --line 10-30 main.go
```

- Edit / apply suggestions inline (if supported):

Some Copilot CLI flows allow you to open an editor or accept suggested edits directly into files. Follow the interactive prompts when the feature is available.

## Example workflow (typical)

1. Open a terminal at the root of your repository
2. Authenticate if needed: `copilot login` or `gh auth login`
3. Run an interactive chat to ask for code suggestions or explanations:

```bash
copilot chat
> Show me how to add a health endpoint that responds with 200/OK and a small JSON body.
```

4. Use `copilot code` to generate files or snippets, then review and commit the generated code.

## Tips and best practices

- Always review and test any generated code before committing — Copilot provides suggestions but they may need adjustments for correctness, style, or security.
- Use the CLI interactively for iterative improvements (ask follow-up questions in `copilot chat`).
- Keep sensitive secrets and credentials out of prompts and repositories.
- Prefer small, focused prompts when generating code: smaller prompts tend to produce more predictable output.

## Troubleshooting

- If the CLI cannot authenticate, try re-running the login flow or using `gh auth login`.
- Check your network/proxy settings if downloads or auth flows fail behind a corporate proxy.
- If a command fails, run with `--help` or `--verbose` (if available) for additional diagnostics.
