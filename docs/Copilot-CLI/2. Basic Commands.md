# Basic Commands

A quick cheat-sheet of the most-used GitHub Copilot CLI commands and in-chat slash commands. Command names and availability can change between releases — run `copilot --help` or `copilot help <command>` for your installed version.

| Command                                                  |         Type | Description                                                                                                                                  | Example                                        |
|----------------------------------------------------------|-------------:|----------------------------------------------------------------------------------------------------------------------------------------------|------------------------------------------------|
| `copilot login` / `/login`                               |   CLI / chat | Authenticate the CLI with your GitHub account (opens browser auth flow).                                                                     | `copilot login`                                |
| `copilot logout`                                         |          CLI | Log out / revoke local CLI auth token.                                                                                                       | `copilot logout`                               |
| `copilot init`                                           |          CLI | Initialize Copilot in a repository or create local config (project onboarding).                                                              | `copilot init`                                 |
| `copilot chat`                                           |          CLI | Start an interactive chat session (REPL).                                                                                                    | `copilot chat`                                 |
| `/help` or `copilot --help`                              |   chat / CLI | Show help text for available commands and flags.                                                                                             | `copilot --help`                               |
| `copilot code`                                           |          CLI | Generate code from a prompt (one-off).                                                                                                       | `copilot code "Create a small Go HTTP server"` |
| `copilot explain`                                        |          CLI | Ask Copilot to explain a file, function or selection.                                                                                        | `copilot explain ./internal/handlers/ping.go`  |
| `copilot --resume`                                       |     CLI flag | Attempt to resume the previous interactive session / conversation.                                                                           | `copilot --resume`                             |
| `copilot --yolo`                                         |     CLI flag | Experimental / aggressive mode (if supported by your version). Use cautiously.                                                               | `copilot --yolo`                               |
| `copilot allow-all` / `allow-all`                        | CLI / config | (Version-dependent) Allow broader suggestion scope or permissive behavior. Check `copilot --help` for exact semantics.                       | `copilot allow-all`                            |
| `copilot clear-history` / `copilot chat --clear-history` |          CLI | Remove local conversation history or instruct CLI to clear stored chats (if supported). Prefer built-in clear commands over manual deletion. | `copilot chat --clear-history`                 |
| `copilot status`                                         |          CLI | Show current auth/status and environment details (if available).                                                                             | `copilot status`                               |
| `copilot version`                                        |          CLI | Print installed copilot version.                                                                                                             | `copilot version`                              |

## In-chat slash commands (use inside `copilot chat`)

When you're already inside the interactive `copilot chat` session you can often use slash commands (type a leading `/`) to control the session, manage authentication, or perform utility actions. Exact slash commands and their behavior vary by Copilot CLI version — type `/help` inside the chat to list the commands supported by your installed version.

| Slash command        | Description                                                                                                                                              | Example                              |
|----------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------|--------------------------------------|
| `/login`             | Start the authentication flow from inside the chat (opens browser or provides a link). Useful if you were prompted to sign in while in the session.      | Type `/login` and follow the prompts |
| `/logout`            | Revoke the locally stored token and log out of the session.                                                                                              | `/logout`                            |
| `/help`              | Show the list of slash commands and short usage notes from within chat. Always run this to see the precise commands supported by your version.           | `/help`                              |
| `/clear` or `/reset` | Clear the current conversation state or start a fresh session. (Behavior may vary: some versions clear the visible messages, others start a new thread.) | `/clear` or `/reset`                 |
| `/resume`            | Attempt to resume the last conversation or continue where you left off. Availability is version dependent.                                               | `/resume`                            |
| `/allow-all`         | (If available) Toggle permissive/allow-all suggestions from inside the chat. Check `/help` text for exact semantics.                                     | `/allow-all`                         |
| `/export`            | Export the current conversation to a file or show an export link (if supported). Use to back up or share chat content.                                   | `/export`                            |
| `/import`            | Import a previously exported conversation into the current session (if supported).                                                                       | `/import <file>`                     |
| `/exit` or `/quit`   | Exit the interactive chat session. Many terminals also accept Ctrl+C to stop the session.                                                                | `/exit` or Ctrl+C                    |

Notes:
- Some commands appear as slash-commands inside the interactive chat (e.g., `/login`, `/clear`) while others are CLI subcommands — availability varies by release.
- If you rely on any of the above commands in scripts or aliases, always verify with `copilot --help` on the machine where the script will run.
- For destructive actions (clearing history, resetting config), prefer built-in commands. If you must delete local files, back them up first.
- Slash commands supported in-chat are implementation-specific. The single reliable action you can take is to run `/help` in your current `copilot chat` session — that will show the exact commands your install supports.
- Some commands (like `/clear`, `/export`, `/import`) can affect local or server-side stored conversation data. Prefer using `/help` and official commands over manual file deletion.
- If a destructive action is offered (e.g., clear or delete history), consider backing up before you proceed.
