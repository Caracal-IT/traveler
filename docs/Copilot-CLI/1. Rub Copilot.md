# Run Copilot

A short cheat-sheet for starting the GitHub Copilot CLI and clearing conversation history.

## Start commands

- Start the default interactive session (REPL/chat):

```bash
copilot
# or explicitly:
copilot chat
```

- Start in an experimental/aggressive mode (if supported by your version):

```bash
copilot --yolo
# or
copilot chat --yolo
```

Use `--yolo` cautiously — it may change behavior to be more permissive or skip confirmations depending on the release.

- Resume the previous session / conversation (if supported):

```bash
copilot --resume
# or
copilot chat --resume
```

- One-off or non-interactive invocations:

```bash
copilot chat "Explain this function"
copilot code "Generate a small Go HTTP server with graceful shutdown"
copilot explain ./path/to/file.go --line 10-30
```

- Get help for flags and commands:

```bash
copilot --help
copilot chat --help
copilot help <command>
```

## Removing chat history

Methods depend on whether history is stored locally or server-side. Always check `copilot --help` first for a built-in clear command.

1) Built-in CLI command (preferred)

Look for an official subcommand or flag to clear history, for example:

```bash
copilot chat --clear-history
# or (examples; availability depends on your installed version):
copilot clear-history
copilot reset
```

2) Remove local cache/config files (if the CLI stores history locally)

Common local locations (macOS / Linux):

- ~/.config/copilot
- ~/.local/share/copilot
- ~/.cache/copilot
- ~/.copilot

Inspect before deleting:

```bash
ls -la ~/.config/copilot
ls -la ~/.local/share/copilot
```

Remove only if you're sure:

```bash
rm -rf ~/.config/copilot
rm -rf ~/.local/share/copilot
rm -rf ~/.cache/copilot
rm -rf ~/.copilot
```

3) Server-side / account-level history

If conversations are stored with your Copilot account, manage or delete them in GitHub settings (look for Copilot or Data/Privacy options):

- https://github.com/settings/copilot

Notes & cautions:

- Prefer a built-in clear command if available; it's safer than deleting files manually.
- Back up any local files before removing them if you might need them later.
- Deleting account/server-side data may be governed by GitHub policies and may not be instant.
