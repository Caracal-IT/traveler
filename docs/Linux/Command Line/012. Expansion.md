# Expansion

Expansion in shell scripting refers to the process of replacing special characters with their corresponding values   
before executing commands. This allows for dynamic and flexible command construction based on variable values or   
other conditions.

## Pathname Expansion

The pathname expansion is a special form of expansion that expands to the value of a variable or a command.
It is commonly used to expand wildcard patterns into matching filenames.
For example, the pattern `*.txt` would expand to all text files in the current directory.
This expansion is particularly useful for file operations and command-line automation.

The * expansion is used to match any sequence of characters.

```bash
exho Press 0 to speak to an agent.
echo Press * to speak to an agent.
```

The first command will print Press 0 to speak to an agent.   
The second command will print Press file1.txt file2.log to speak to an agent.  

```bash
ls *.html
cat blue*
le -l *at*
```

## The ? Wildcard

The ? wildcard matches any single character.
It is useful for matching filenames with a specific number of characters.
For example, the pattern `file?.txt` would match `file1.txt`, `file2.txt`, but not `file12.txt`.

```bash
ls file?.txt
echo *.???
```

## Range Wildcard

The range wildcard is used to match a range of characters.
It is typically used to match a sequence of numbers or letters.
For example, the pattern `[0-9]` would match any single digit, `[a-z]` would match any single lowercase letter,
and `[A-Z]` would match any single uppercase letter. 1-9, A-Z, a-z are all valid ranges.
For example, the pattern `[a-c]` would match `a`, `b`, or `c`. You can negate a range by prefixing it with a ^.   
For example, the pattern `[^0-9]` would match any character that is not a digit. 

```bash
echo [a-c]*.???
```

## The Tilde (~) Expansion

The tilde expansion is used to expand to the home directory of the current user.  
It is commonly used to simplify paths to the user's home directory.  When used alone, it expands to the current   
user's home directory. When used with a slash, it expands to the home directory of the specified user. If the user   
does not exist, it expands to nothing.  When used with ~user, it expands to the home directory of the specified user.

```bash
cd ~ 
```

## The Magic of Brace Expansion

Brace expansion is a feature that allows you to generate multiple filenames or strings from a single pattern.  
It is useful for creating multiple files or directories with similar names.  The pattern is enclosed in curly braces,  
and the elements are separated by commas.  For example, the pattern `{file1,file2,file3}` would expand to `file1`, `file2`, and `file3`.  
You can also use ranges in brace expansion.  For example, the pattern `{file{1..3}}` would expand to `file1`, `file2`, and `file3`.  
You can also use nested brace expansion.  For example, the pattern `{file{1..3},dir{1..3}}` would expand to `file1`, `file2`, `file3`, `dir1`, `dir2`, and `dir3`.  
You can also use brace expansion with wildcards.  For example, the pattern `{file{1..3}*.txt}` would expand to `file1.txt`, `file2.txt`, and `file3.txt`.  
You can also use brace expansion with variables.  For example, the pattern `{${var1},${var2}}` would expand to the values of `var1` and `var2`.
You can also crete nested brace expansion.  For example, the pattern `{file{1..3},dir{1..3},subdir{1..3}}` would expand to `file1`, `file2`, `file3`, `dir1`, `dir2`, `dir3`, `subdir1`, `subdir2`, and `subdir3`.

```bash
echo {2..10..2} 
```

## Arithmetic Expansion

The arithmetic expansion is used to perform arithmetic operations on variables or strings.  It is commonly used to   
generate filenames or strings based on a sequence of numbers.  The pattern is enclosed in parentheses and consists of     
arithmetic operators and variables. Only integer arithmetic is supported.

The syntax is `$((arithmetic-expression))` 

We can write arithmetic expressions using:
- `+` for addition
- `-` for subtraction
- `*` for multiplication
- `/` for division
- `%` for modulo (remainder)
- `**` for exponentiation


For Example,    
1. The pattern `(1..10)` would expand to `1`,
2. The pattern `(1..10..2)` would expand to `1`, `3`, `5`, `7`, and `9`.
3. The pattern `$(1..10..2)` would expand to `1`, `3`, `5`, `7`, `9`.
4. The pattern `$(($((1+2))) + 3)` would expand to `5`.

## Double VS Single Quotes

The difference between single and double quotes is that single quotes do not interpret any special characters. 
For example, the pattern `'Hello, $USER!'` would expand to `'Hello, $USER!'`, while the pattern `"Hello, $USER!"`   
would expand to `"Hello, username!"` if the variable `USER` is set to `username`. Special characters like `$`, `\`,   
``` and `*` are not expanded in single quotes.  

## Command Substitution

We can use the command substitution to execute a command and capture its output.  The syntax is `$(command)`.  


```bash
echo "Today is $(date +%A)"
```

The output of the command `date +%A` will be captured and assigned to the variable `Today`.
