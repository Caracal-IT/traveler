# Expansion

Expansion in shell scripting refers to the process of replacing special characters with their corresponding values   
before executing commands. This allows for dynamic and flexible command construction based on variable values or   
other conditions.

## Pathname Expansion

The pathname expansion is a special form of expansion that expands to the value of a variable or a command.
It is commonly used to expand wildcard patterns into matching filenames.
For example, the pattern `*.txt` would expand to all text files in the current directory.
This expansion is particularly useful for file operations and command-line automation.

The * expansion is used to match any sequence of characters.

```bash
exho Press 0 to speak to an agent.
echo Press * to speak to an agent.
```

The first command will print Press 0 to speak to an agent.   
The second command will print Press file1.txt file2.log to speak to an agent.  

```bash
ls *.html
cat blue*
le -l *at*
```

## The ? Wildcard

The ? wildcard matches any single character.
It is useful for matching filenames with a specific number of characters.
For example, the pattern `file?.txt` would match `file1.txt`, `file2.txt`, but not `file12.txt`.

```bash
ls file?.txt
echo *.???
```

## Range Wildcard

The range wildcard is used to match a range of characters.
It is typically used to match a sequence of numbers or letters.
For example, the pattern `[0-9]` would match any single digit, `[a-z]` would match any single lowercase letter,
and `[A-Z]` would match any single uppercase letter. 1-9, A-Z, a-z are all valid ranges.
For example, the pattern `[a-c]` would match `a`, `b`, or `c`. You can negate a range by prefixing it with a ^.   
For example, the pattern `[^0-9]` would match any character that is not a digit. 

```bash
echo [a-c]*.???
```
